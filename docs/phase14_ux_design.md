# Phase 14: カレンダー・リスト操作のUX改善設計

## 1. 概要
モバイル環境での操作性を向上させるため、カレンダー画面やリスト操作周辺で発生している以下のUI/UX上の課題を解決します。
1. 横スワイプによる月切り替えの誤爆（縦スクロール時に意図せず切り替わる）
2. レシート一覧のスクロール中に、長押し判定が暴発して編集画面が開いてしまう
3. モバイル表示時、カレンダーのセルの縦幅が大きく、レシート一覧の表示領域が狭くなっている問題

## 2. 解決策と実装変更点

### 2-1. 横スワイプ誤爆の防止
- **変更対象**: `src/routes/calendar/+page.svelte`
- **想定する変更**:
  - `touchstart` 時に X座標に加え Y座標（`screenY`等）も記録します。
  - `touchend` 時に X軸の移動量（横方向）と Y軸の移動量（縦方向）の絶対値を比較。
  - **縦方向の移動量が横方向の移動量を上回っている場合（意図的な縦スクロール）は、スワイプ判定をキャンセル**します。
  - その他、スワイプと判定する閾値（現在50px）を様子を見て若干調整します。

### 2-2. スクロール時の長押し（Longpress）暴発防止
- **変更対象**: `src/lib/utils/longpress.ts`
- **想定する変更**:
  - `mousedown` / `touchstart` 送出時に、開始座標を保存します。
  - 新たに `mousemove` および `touchmove` イベントリスナーを追加します。
  - ポインターが移動した際、開始位置から一定量（閾値例: `10px`）動いた場合は「スクロール」または「ドラッグ操作」とみなし、長押しタイマー（`setTimeout`）を強制的にキャンセルします。

### 2-3. カレンダーセルの高さコンパクト化
- **変更対象**: `src/lib/components/calendar/CalendarGrid.svelte`
- **想定する変更**:
  - コンポーネントの `<style>` で `.day-cell` に適用されている `min-height: 4.5rem;` を、表示内容が安全に収まる範囲でコンパクト化します（例：`3.5rem` 〜 `4rem` 程度に削減、上下の余白微調整を含む）。
  - カレンダー全高を下げ、下部のレシート一覧コンテナの表示面積を拡大します。

## 3. 検証プラン
- 実機サイズのブラウザ（DevToolsデバイストグル表示）で以下を確認：
  - カレンダー画面上で上下にスワイプ（スクロール）した際、月が予期せず切り替わらないこと。
  - レシート一覧をゆっくり指でなぞってスクロールした際に、長押しイベントが発火（=編集画面へ遷移）しないこと。
  - カレンダーの高さが適度に抑えられ、従来のレイアウト情報を損なわずに一覧領域が広くなっていること。
